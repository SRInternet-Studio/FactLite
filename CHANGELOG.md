# Changelog

## [2.0.0-alpha1] - 2026-05-24

### 新增功能

- **Web_LLMJudge 升级**：
  - 新增 QueryRewriter（查询重写模块），自动分词提取关键词生成搜索词
  - 实现 N-Gram 重排方案，使用 Bi-gram Jaccard 相似度算法提取最相关内容
  - 支持 jieba 分词库进行中文分词
  - 支持自动意图检测（Auto Route），智能判断是否需要联网搜索
  
- **新增 Augmenter（联网增强生成）**：
  - 事前智能检索增强器，自动判断用户意图
  - 支持同步和异步两种调用方式
  - 整合搜索、重排序、Prompt 组装为一体
  - 可直接通过 `from FactLite import Augmenter` 使用

- **语义重排模块**：
  - 基于 Sentence Transformers 的本地重排器
  - 支持自定义模型和相似度阈值
  - 提高搜索结果相关性

- **代码结构优化**：
  - 在 `core` 目录下创建 `web_utils` 子目录

### 改进

- **异步支持增强**：优化异步执行流程，使用 `asyncio.to_thread` 处理阻塞操作
- **意图检测算法**：结合模式匹配和 LLM 判断，提高检测准确率
- **日志系统**：统一日志配置，支持多级别日志输出

## [1.2.0] - 2026-04-18

### 新增功能

- **规则链 (Rule Chaining)**：支持顺序执行多个验证器，创建复杂的验证工作流程
- **RegexValidator 增强**：支持从 TXT 文件加载违禁词汇
- **新增 LengthValidator**：验证回复长度，支持包含/排除标点符号
- **新增 ModerationJudge**：使用 OpenAI Moderation API 检测不安全内容
- **no_retry 支持**：当验证失败是由非 AI 生成内容导致时，可直接执行兜底策略

### 改进

- **参数名统一**：将 `rule` 参数重命名为 `rules`，保持一致性
- **代码精简**：优化了 rules.py 文件的代码结构，使用更高级的语法
- **错误处理**：为所有验证器添加了异常处理和 no_retry 标记
- **文档更新**：更新了 README.md 和 README_CN.md，添加了新功能的说明

### 修复

- 修复了 API 密钥失效导致框架崩溃的问题
- 修复了验证器异常处理的问题

## [1.1.0] - 2026-03-15

### 新增功能

- **Web_LLMJudge**：支持联网搜索增强验证
- **自定义裁判接口**：支持用户定义自己的评估规则
- **异步支持**：支持异步函数，避免装饰器堵塞主线程
- **模块化设计**：将代码拆分为多个文件和子目录

### 改进

- 优化了验证流程，提高了框架的稳定性
- 增加了异常处理，确保框架不会因 API 错误而崩溃
- 改进了文档，添加了更多使用示例

## [1.0.0] - 2026-02-01

### 初始版本

- 实现了基本的 AI 自我反思和事实核查功能
- 支持 OpenAI API 进行评估
- 提供了多种失败处理策略
- 支持装饰器模式，易于集成到现有代码中
